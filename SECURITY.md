# Security Policy

## Supported Versions

Only the latest tagged release is supported with security updates.

## Reporting a Vulnerability

If you discover a security vulnerability, do not open a public issue.

Send a report to the support contact from your purchase receipt and include:
- affected version
- steps to reproduce
- impact assessment
- any suggested fix (optional)

## Scope Notes

This starter kit includes security-focused features (MFA, session revoke, IP allowlist, audit logs), but you are still responsible for:
- secure deployment configuration (HTTPS, secrets, backups)
- infrastructure security
- ongoing dependency updates
